# CHAPTER 1
The Early Software-Sharing Community

    In the 1970s, software was freely shared at MIT’s AI Lab, where Stallman worked.
    The Incompatible Timesharing System (ITS) allowed users to modify and improve programs.
    However, the rise of proprietary software in the 1980s dismantled this culture, restricting users’ ability to share or modify software.

The Fall of Software Freedom

    Proprietary software led to user restrictions and secrecy.
    Stallman saw this as morally wrong, forcing users into a system where they had to beg for software changes.
    He rejected working in the proprietary industry and instead created GNU, a free operating system.

The GNU Operating System

    The GNU system aimed to be a complete, Unix-compatible OS but with free software principles.
    GNU software includes compilers, text editors, shells, and utilities.

Copyleft and the GNU General Public License (GPL)

    To ensure software remains free, Stallman introduced copyleft—using copyright to prevent proprietary restrictions.
    The GPL ensures that software and its modifications remain free for everyone.

Challenges to Free Software

    Secret hardware: Companies refuse to share hardware specs, making free driver development difficult.
    Non-free libraries: Proprietary libraries (e.g., Qt, Motif) trap developers into creating dependent non-free software.
    Software patents: Legal barriers that prevent free software from implementing certain features.
    Lack of free documentation: Most software manuals remain proprietary, limiting education.

The Free vs. Open Source Debate

    The term "open source" emerged in 1998 to make free software more business-friendly.
    Stallman argues that this shift dilutes the ethical and freedom-based mission of GNU.

Conclusion

    Stallman urges developers to prioritize freedom over convenience.
    The free software movement must continue to defend users' rights against proprietary control.
    
    
# CHAPTER 2
    What is GNU?
        GNU (Gnu’s Not Unix) is a free software project that provides a complete OS with compilers, text editors, and utilities.
        Unlike Unix, GNU ensures users can modify and share the software.

    Why Free Software?
        Stallman believes proprietary software is unethical because it restricts users' ability to cooperate.
        The Golden Rule: If he likes a program, he should share it with others.

    How GNU is Distributed
        Not public domain, but free under the GPL, ensuring no one can relicense it as proprietary.

    Why Programmers Support GNU
        Many programmers dislike proprietary software because it forces competition over cooperation.
        GNU provides an alternative where collaboration is encouraged.

    Benefits of GNU for All Users
        No license fees, reducing software costs.
        Open source code, allowing users to modify and improve software.
        Better security and reliability since users can inspect the code.

    Common Objections to Free Software
        "No one will support free software" → Companies can still sell support services.
        "No one will program for free" → Many programmers already contribute for ethical and personal reasons.
        "Programmers won’t make money" → Alternative funding models include donations, subscriptions, and service-based income.

    Final Thoughts
        Free software is not about eliminating profits—it’s about ensuring freedom and user control over technology.
        Society benefits when software is shared and improved collectively, rather than locked behind corporate paywalls.

Both chapters emphasize that freedom in software is a moral issue, not just a practical concern.
